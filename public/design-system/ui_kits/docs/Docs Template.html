Quickstart

Serve your first Boost Boss ad in under five minutes — pick an integration door, drop in your key, and render an intent-matched placement.

★ Start hereUpdated Jun 2026·4 min read

Boost Boss serves contextual, intent-matched ads inside AI applications. This guide takes you from zero to a live placement using the JS snippet — the fastest door. Prefer a typed SDK or raw HTTP? Jump to the npm SDK or REST API.

You'll need a publisher API key. Generate one in the dashboard under Settings → API keys. Keys are scoped per surface.

1. Install

Add the Boost Boss client to your project. The same package powers every surface — browser, computer, extension, and mobile.

// install via npm
import { BoostBoss } from '@boostbossai/sdk';

const bb = new BoostBoss({ apiKey: 'bb_pk_live_***' });
// install via npm
import { BoostBoss, AdRequest } from '@boostbossai/sdk';

const bb: BoostBoss = new BoostBoss({ apiKey: 'bb_pk_live_***' });
# pip install boostboss
from boostboss import BoostBoss

bb = BoostBoss(api_key="bb_pk_live_***")
# no install — call the REST API directly
export BB_KEY="bb_pk_live_***"

2. Request an ad

Pass the live conversation context. Benna decodes intent and returns the highest-matched placement in ~5ms.

const ad = await bb.fetch({
  context: userMessage,
  surface: 'browser',
  format: 'native'
});

render(ad);
const ad: AdRequest = await bb.fetch({
  context: userMessage,
  surface: 'browser',
  format: 'native'
});

render(ad);
ad = bb.fetch(
    context=user_message,
    surface="browser",
    format="native",
)

render(ad)
curl https://api.boostboss.ai/v1/ads \
  -H "Authorization: Bearer $BB_KEY" \
  -d '{"context":"deploy a next.js app","surface":"browser"}'

Tip: Pass as much real conversation context as you can. Benna's match quality scales directly with signal — richer context means higher CTR and eCPM.

3. Handle the response

Every ad request returns a typed payload. Render the creative and report the impression — the SDK fires tracking automatically when you call render().

Request parameters

ParameterTypeRequiredDescription
contextstringRequiredThe live user message or conversation snippet Benna decodes for intent.
surfaceenumRequiredOne of browser, computer, extension, mobile.
formatenumOptionalPlacement format. Defaults to native.
bb_clickstringOptionalAffiliate click ID for attribution pass-through.

Endpoint

POSThttps://api.boostboss.ai/v1/ads
MethodEndpointReturns
POST/v1/adsAn ad placement matched to the supplied context.
GET/v1/ads/:idRetrieve a previously served placement and its status.
DELETE/v1/ads/:idVoid an unrendered placement before it bills.

Heads up: Never expose a secret key (bb_sk_*) in client-side code. Browser and extension surfaces must use a publishable key (bb_pk_*) and proxy sensitive calls through your backend.
