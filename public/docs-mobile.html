Lumi for Mobile App

The highest-CPM ad format in adtech, one command away.

For iOS and Android apps — whether you build with React Native, Swift, or Kotlin. One CLI auto-detects which kind of project you're in and wires the right SDK underneath: native iOS (Swift/Obj-C via CocoaPods + SPM), native Android (Kotlin/Java via Gradle + Maven), or the React Native wrapper that bridges to both natives.

Install

npx @boostbossai/install-mobile YOUR_PUBLISHER_ID

What the CLI does:

  • Auto-detects your project type — Xcode project → iOS native flavor, Gradle project → Android native flavor, RN package.json → React Native flavor
  • For React Native: runs npm install @boostbossai/lumi-mobile, wraps your root component's outermost JSX with <LumiProvider publisherId="...">, and on Mac runs cd ios && pod install
  • For native iOS: adds pod 'BoostBossLumi' to your Podfile (or Swift Package Manager URL if you use SPM) and inserts the SDK init in AppDelegate
  • For native Android: adds implementation 'ai.boostboss:lumi:<version>' to your build.gradle.kts and inserts the SDK init in Application.onCreate
  • Writes .lumi-install-log so --uninstall can reverse everything

If auto-detect picks wrong, force the flavor with --platform=ios, --platform=android, or --platform=rn.

Rebuild your app (npx expo run:ios / npx react-native run-ios / Xcode / Android Studio). The Mobile App verify badge on your dashboard flips to Connected within 60 seconds.

Why three flavors instead of one

SKAdNetwork (iOS) and Play Install Referrer (Android) are the standard install-attribution APIs every mobile ad network has to hit — and they're only callable from native code. JavaScript can't reach them. AppLovin, Unity, and IronSource all ship the same shape: native iOS SDK + native Android SDK + thin wrappers on top for React Native, Unity, Flutter, and Cordova. Lumi follows that pattern. Your React Native install gets the same underlying attribution stack as a native Swift install, because the RN package links the native SDKs at build time.

Rewarded video — the premium unit

The <BottomBanner /> auto-mounts when LumiProvider sees your app. The premium <RewardedVideo /> is opt-in — drop it anywhere a user would benefit from earning an in-app reward:

import { RewardedVideo } from '@boostbossai/lumi-mobile';

<RewardedVideo
  buttonLabel="Watch ad for 10 credits"
  onReward={(reward) => grantUserCredits(10)}
/>

Rewarded video is the highest-CPM ad format in all of adtech ($25–50 CPM). Users opt in explicitly, watch a 15–30 second video, and you grant them their reward. Apple and Google App Store policies fully support this format — it's how AppLovin and Unity print money.

Compatibility

Covers:

  • React Native — Expo SDK 50+ (managed workflow), bare RN 0.74+
  • Native iOS — Swift 5.5+ / Obj-C, iOS 14+, distributed via CocoaPods and Swift Package Manager
  • Native Android — Kotlin / Java, API 23+ (Android 6.0+), distributed via Maven Central and JitPack
  • Architectures: iOS arm64 + iOS Simulator (arm64 + x86_64), Android arm64-v8a + armeabi-v7a + x86_64

Does not cover (yet):

  • Flutter — track the issue at our GitHub if you'd like to be notified when it ships
  • Unity — no roadmap; build complexity outweighs current AI-app overlap
  • Cordova / Capacitor / Ionic — use the Browser App SDK in the WebView shell instead, it works there
  • iOS < 14, Android < 6.0 — outside SKAdNetwork / modern attribution support windows

If your stack isn't listed and you'd like it supported, email support@boostboss.ai. We prioritize SDK additions by publisher request volume.

Uninstall

npx @boostbossai/install-mobile --uninstall

The CLI reads .lumi-install-log, reverses every patch (unwraps the LumiProvider, removes from Expo plugins, uninstalls the npm package), deletes the log.

Privacy posture

App-session scoped · no cross-app tracking · App Store / Play Store compliant.

Session UUIDs are generated per app launch and discarded on close. No IDFA, no Google Advertising ID, no fingerprinting. Rewarded video is opt-in by user click (Apple guideline 4.5.4 explicit allowlist). No background data collection. Full privacy policy.
