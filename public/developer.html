1. Install

npm install @boostbossai/lumi-mcp

2. Wire it into a tool handler

// Inside your existing MCP server's tool handler
import { LumiMCP } from '@boostbossai/lumi-mcp';

const lumi = new LumiMCP({
  publisherId: process.env.BB_PUBLISHER_ID,
  apiKey:      process.env.BB_API_KEY,
});

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const result = await myToolLogic(request.params.arguments);

  const ad = await lumi.fetchAd({
    context:  request.params.name,
    toolName: request.params.name,
  });

  return {
    content: ad
      ? [...result.content, ad.toMCPBlock()]   // appended, never replaces
      : result.content,
  };
});
toMCPBlock() bakes the disclosure label in. Try sandbox without signup: pass publisherId: "pub_test_demo". Full reference →

1. Drop into your <head>

<script async
  src="https://boostboss.ai/lumi.js"
  data-publisher-id="bb_dev_sk_loading"></script>
Async, zero impact on first paint. The publisher ID above is your live key — already filled in.

2. Add slot divs where you want ads

<div data-lumi-slot="banner"></div>
<div data-lumi-slot="sidebar" data-lumi-context="checkout flow"></div>
<div data-lumi-slot="inline"></div>
Lumi auto-discovers slots and rerenders on SPA route changes via MutationObserver. Theme via CSS variables (--lumi-primary, --lumi-radius, etc.). Full reference →

1. Install

npm install @boostbossai/lumi-sdk

2. Render in your content script / sidepanel

import { Lumi } from '@boostbossai/lumi-sdk';

const lumi = new Lumi({ publisherId: 'bb_dev_sk_loading' });

await lumi.render('#ad-slot', {
  format:  'sidebar',
  context: 'user is reading docs',
});

lumi.on('impression', (e) => console.log('shown', e.adId));
lumi.on('error',      (e) => console.warn(e.code, e.message));

2b. React or Vue?

// React
import { LumiProvider, LumiSlot } from '@boostbossai/lumi-sdk/react';

<LumiProvider publisherId="bb_dev_sk_loading">
  <LumiSlot format="sidebar" context="user is reading docs" />
</LumiProvider>

// Vue 3
import { provideLumi, LumiSlot } from '@boostbossai/lumi-sdk/vue';
provideLumi({ publisherId: 'bb_dev_sk_loading' });
// <LumiSlot format="sidebar" :context="docTitle" />
Manifest v3 native — zero eval, zero remote code. Allow https://boostboss.ai in your extension's CSP. Full reference →

1. Add bearer token to env

BB_API_KEY=bb_dev_sk_loading
Server-side only — never ship the bearer to the browser. Use Vercel / Netlify / Render env-var settings.

2. Direct REST call (Discord example)

// Inside your Discord bot's response handler:
const { ad } = await fetch('https://boostboss.ai/v1/ad-request', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.BB_API_KEY}`,
    'Content-Type':  'application/json',
  },
  body: JSON.stringify({
    format:   'embed',
    context:  userQuery,
    platform: 'discord',
  }),
}).then(r => r.json());

if (ad) {
  await message.reply({ content: aiResponse, embeds: [/* your formatting */] });
  fetch(ad.impression_url).catch(() => {});  // fire-and-forget
}

2b. Optional: per-platform helper library

// Discord — convenience wrappers around the Lumi API for Bots
npm install @boostbossai/lumi-discord
// or:  @boostbossai/lumi-telegram  ·  @boostbossai/lumi-slack

import { toDiscordEmbed, toDiscordComponents } from '@boostbossai/lumi-discord';

await message.reply({
  content:    aiResponse,
  embeds:     [toDiscordEmbed(ad)],
  components: [toDiscordComponents(ad)],
});
Helper libraries are optional — they format the ad payload into each platform's native message type so you don't have to. Full reference →

Quick Start

                

                    
                    
const bb = require("@boostbossai/lumi-sdk");

// 1. Authenticate so impressions are attributed to your account.
bb.configure({ apiKey: process.env.BB_API_KEY });

// 2. Request a Benna-ranked ad for the current user context.
const ad = await bb.getSponsoredContent({
  context: "user is debugging a python traceback",
  host:    "your-app.com",
  format:  "native"      // "image" | "native"
});

// 3. Render and track. SDK fires the impression beacon for you.
if (ad.sponsored) {
  showAd(ad.sponsored);
  bb.trackEvent("impression", ad.sponsored.id);
}
