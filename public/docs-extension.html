Lumi for Browser Extension App

One terminal command. 8 placements.

For Chrome / Edge / Brave / Opera browser extensions (Manifest V3). The install CLI patches your manifest.json, drops the SDK into your service worker, and wires popup + side panel + new-tab pages where they exist.

Install

From your extension's project root:

npx @boostbossai/install-extension YOUR_PUBLISHER_ID

What the CLI does:

  • Runs npm install @boostbossai/lumi-extension
  • Adds "storage" to your permissions
  • Adds "https://boostboss.ai/*" to your host_permissions
  • Prepends a 3-line init block to your background service worker
  • Adds a <script> tag to your popup.html / sidepanel.html / newtab.html if those exist
  • Writes a .lumi-install-log recording every change for symmetric uninstall

Reload your extension in chrome://extensions. The Browser Extension verify badge on your dashboard flips to Connected within 60 seconds.

Manifest V3 only

The Chrome Web Store has been Manifest V3-only since 2024. If your extension is still on V2, migrate first (Chrome's guide) and re-run the installer. The CLI errors out on V2 with a link to the migration guide.

The runtime is bundled into your extension's node_modules/@boostbossai/lumi-extension/dist/ at install time — no remote code loading, no eval, fully Manifest V3 compliant.

Firefox / Safari

v1.2 supports Chrome / Edge / Brave / Opera (Chromium MV3). Firefox port lands in v1.3. Safari Web Extensions are a separate effort (App Store distribution required) — v1.4. Track progress at your dashboard.

Uninstall

npx @boostbossai/install-extension --uninstall

The CLI reads .lumi-install-log, reverses every patch (manifest, service worker, HTML files), uninstalls the npm package, deletes the log. Your extension is back to its pre-install state.

Privacy posture

Active-tab content only · session-storage scoped to your extension.

The runtime reads the URL of the active tab plus your extension's own UI surfaces (popup / sidepanel / newtab). Nothing is read from other tabs unless you've granted the extension cross-tab permissions yourself. Session UUIDs are stored in chrome.storage.session (auto-cleared on browser restart). No localStorage, no cookies, no cross-site tracking.
