React wrapper for Lumi

The bare <script> tag from the main install doc works fine in React, Next.js, and every other framework — paste it into index.html or app/layout.tsx. This wrapper is optional.

Why we don't ship this as an npm package.

Two install paths means two ways to fail, two sets of docs, and two FAQs. We chose one obvious way. If you'd rather wrap the script tag in a component, copy the 12 lines below into your project. We don't ship or version an npm package — you control the wrapper.

The wrapper

Copy this into components/LumiScript.tsx (TypeScript) or components/LumiScript.jsx (plain JS):

// components/LumiScript.tsx
import { useEffect } from 'react';

export function LumiScript({ publisherId }: { publisherId: string }) {
  useEffect(() => {
    if (document.querySelector('script[data-lumi]')) return;
    const s = document.createElement('script');
    s.src = `https://boostboss.ai/lumi/v1.js#${publisherId}`;
    s.async = true;
    s.setAttribute('data-lumi', 'browser-app');
    document.head.appendChild(s);
  }, [publisherId]);
  return null;
}

Use it

Mount it once at your app root — the layout component for App Router, _app.tsx for Pages Router, or the root of your component tree for SPA setups:

// app/layout.tsx (Next.js App Router)
import { LumiScript } from '@/components/LumiScript';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <LumiScript publisherId={process.env.NEXT_PUBLIC_LUMI_ID!} />
        {children}
      </body>
    </html>
  );
}

Notes

  • Guard against double-mount. The data-lumi attribute check prevents Next.js Strict Mode from injecting the script twice during dev. In production this is a no-op.
  • Pass the publisher ID via env var. Use NEXT_PUBLIC_LUMI_ID (Next.js), VITE_LUMI_ID (Vite), or your framework's public-env convention. Don't hardcode it into the component.
  • Privacy posture unchanged. The wrapper imposes no extra data collection — it just mounts the same script tag the main doc shows. See privacy policy.
  • You own this wrapper. If Lumi ships a v2 with new options, you decide when to update it. We don't push npm version bumps that surprise you.

What about Server Components?

The wrapper above is a Client Component (it uses useEffect). That's intentional — the script tag must mount on the client to access window and document. If you're in a Server Component context, render the script tag directly via Next.js's <Script> component:

import Script from 'next/script';

<Script
  src={`https://boostboss.ai/lumi/v1.js#${process.env.NEXT_PUBLIC_LUMI_ID}`}
  strategy="afterInteractive"
/>
